## 1. ОБЩАЯ СХЕМА ПАЙПЛАЙНА

```
[БРИФ от клиента] → [✅ Согласование брифа]
    → [Сбор источников] → [Текст лекции] → [✅ Согласование текста с клиентом]
```

## ЭТАП 1: ПОДГОТОВКА БАЗЫ ЗНАНИЙ (в Gemini)

### Шаг 1. Брифинг темы (из заполненного брифа)
Зафиксировать:
- **Тема:** (например: "Основы электродуговой сварки")
- **ЦА:** (например: "Начинающие сварщики, рабочие производства")
- **Главная боль:** (например: "Высокий процент брака из-за незнания базовых режимов сварки")
- **Длительность итогового видео:** (рекомендуем 8–15 минут на лекцию)
- **Источники от клиента:** (регламенты, ГОСТы, инструкции — то, что дал клиент в брифе)

### Шаг 2. Промт на поиск источников (в Gemini)
```

на русском языке из России.
```

### Шаг 3. Сбор источников
Скачать PDF, txt, Google Docs. **Минимум 3 качественных источника.**

### Шаг 4. Методологический промт (в Gemini)

## ЭТАП 2: СОЗДАНИЕ ТЕКСТА ЛЕКЦИИ (в NotebookLM или Gemini)

### Шаг 5. Генерация текста лекции
Используем тот же процесс "чистой комнаты":
- Новый блокнот в NotebookLM
- Загрузить `00_SYSTEM_RULES.txt` + источники
- Генерировать по одной лекции

### Шаг 6. QA текста
Прогнать через промт проверки (файл `промт для проверки текста лекций.docx`):
- Смысл и логика
- Практическая польза
- Борьба с канцеляритом
- Структура

**Результат этапа:** Чистый, проверенный текст лекции.

### Точка согласования №2: ТЕКСТ ЛЕКЦИИ
- [ ] Отправить текст клиенту на проверку (Google Docs или .docx)

## ЭТАП 3: АДАПТАЦИЯ В ВИДЕОСЦЕНАРИЙ (Новый этап!)

> Это ключевой этап, которого нет в NotebookLM-процессе. Здесь текст лекции превращается в режиссёрский сценарий.

### Шаг 7. Разбивка на сцены

Каждая сцена = 30–90 секунд экранного времени.
Типы сцен:

| Тип сцены | Что на экране | Когда использовать |
|-----------|---------------|-------------------|
| **АВАТАР** | Лектор говорит на камеру (крупный план) | Введение, объяснение концепций, выводы |
| **АВАТАР + СЛАЙД** | Лектор в углу + презентация на фоне | Теория, схемы, таблицы, формулы |
| **B-ROLL** | Видеовставка + закадровый голос | Демонстрация процесса, примеры, визуализация |
| **СЛАЙД** | Только слайд + закадровый голос | Сложные схемы, где аватар будет мешать |

### Шаг 8. Промт для создания сценария (в Gemini)

```
Ты — режиссёр образовательных видеолекций.

Вот текст лекции:
[ВСТАВИТЬ ТЕКСТ ЛЕКЦИИ]

Преврати его в видеосценарий по следующим правилам:

1. Разбей на сцены по 30–90 секунд (примерно 80–200 слов на сцену)
2. Для каждой сцены укажи:
   - Номер сцены
   - Тип: АВАТАР / АВАТАР+СЛАЙД / B-ROLL / СЛАЙД
   - Текст диктора (адаптированный для устной речи)
   - Визуал: что показать на слайде или какой B-roll нужен
   - [Для B-ROLL] Промт для генерации видео на английском языке

3. Правила адаптации текста для диктора:
   - Разбей длинные предложения (макс. 15–20 слов)
   - Убери скобки, сноски, нумерацию типа "п. 3.2.1"
   - Добавь связки между сценами ("Теперь посмотрим...", "А вот здесь важный момент...")
   - Числа и аббревиатуры — расшифруй ("ГОСТ" → "гост", "150°С" → "сто пятьдесят градусов")
   - НЕ ДОБАВЛЯЙ информацию, которой нет в тексте лекции!

4. Структура видео:
   - Сцена 1: АВАТАР — Приветствие + о чём лекция (15–20 сек)
   - Сцены 2–N: Основной контент (чередуй типы сцен!)
   - Последняя сцена: АВАТАР — Итоги + что запомнить (20–30 сек)

5. Соотношение сцен (примерное):
   - АВАТАР: 30%
   - АВАТАР+СЛАЙД: 40%
   - B-ROLL: 20%
   - СЛАЙД: 10%

Выдай результат в виде таблицы.
```

## ЭТАП 4: ПОДГОТОВКА МАТЕРИАЛОВ

### Шаг 10. Создание презентации (.pptx)

Для всех сцен типа АВАТАР+СЛАЙД и СЛАЙД — подготовить слайды.

**Правила оформления слайдов для HeyGen:**
- Минимум текста на слайде (заголовок + 3–5 буллетов или схема)
- Крупный шрифт (мин. 24pt)
- Единый стиль для всего курса (шаблон М-Платформы)
- Правая или левая часть слайда оставить свободной для аватара (если формат АВАТАР+СЛАЙД)
- Формат: 16:9, .pptx

### Шаг 11. Подготовка B-roll материалов

Для каждой B-ROLL сцены — один из вариантов:
1. **AI-генерация в HeyGen** — использовать встроенный генератор (Google Veo 3.1, Sora 2, Seedance). Промты на английском.
2. **Стоковое видео** — из встроенной библиотеки HeyGen или внешних стоков.
3. **Собственное видео** — если у клиента есть съёмка производства, процессов и т.д.

> **Для курса по сварке:** идеально попросить у клиента видео с их производства. Это повышает ценность в разы.

### Точка согласования №3: СЦЕНАРИЙ + ВИЗУАЛ
- [ ] Отправить клиенту таблицу сценария (какие сцены, что говорит диктор, что на экране)
- [ ] Отправить превью слайдов (PDF или скриншоты)
- [ ] Показать выбранного аватара (скриншот или 15-секундное тестовое видео)
- [ ] Дать послушать выбранный голос (тестовая фраза)
- [ ] Клиент подтвердил: сценарий + визуал + аватар + голос (письменно)

> **Зачем:** После этого этапа начинается рендер, который стоит credits. Переделка = двойная трата. Согласование сценария — последний шанс внести правки "бесплатно".

## ЭТАП 5: СБОРКА В HEYGEN

### Шаг 12. Создание проекта

1. Зайти в HeyGen → **AI Studio** → Create Video
2. Выбрать формат **16:9** (ландшафт)
3. Язык: **Русский**

### Шаг 13. Выбор аватара

**Варианты:**
- **Стоковый аватар** — из библиотеки HeyGen (100+ вариантов). Выбрать профессионально выглядящего.
- **Кастомный аватар клиента** — если клиент хочет "своего" лектора (требуется 2–5 мин видео человека, обработка 5–7 дней).
- **Фото-аватар** — из фотографии (быстро, но менее качественно).

**Рекомендация для стандартных лекций:** Использовать один и тот же аватар для всего курса. Выбрать аватар, который выглядит как эксперт (не слишком молодой, деловой стиль).

### Шаг 14. Выбор голоса

- Зайти в раздел **Voices**
- Выбрать русскоязычный голос (есть несколько вариантов мужских и женских)
- Прослушать превью, выбрать наиболее естественный
- **Опционально:** клонировать голос реального лектора (нужен образец записи)

> Зафиксировать выбранный голос и аватар для всего курса!

### Шаг 15. Сборка сцен (по сценарию из Шага 8–9)

Идём строго по таблице сценария, сцена за сценой:

**Для сцены типа АВАТАР:**
1. Добавить новую сцену
2. Поставить аватар по центру или крупным планом
3. Вставить текст диктора в поле скрипта
4. Фон: нейтральный (можно брендированный)

**Для сцены типа АВАТАР+СЛАЙД:**
1. Добавить новую сцену
2. Загрузить слайд как фон (или использовать PPT-to-Video)
3. Разместить аватар в углу (picture-in-picture, ~25% экрана)
4. Вставить текст диктора

**Для сцены типа B-ROLL:**
1. Добавить новую сцену
2. **БЕЗ аватара** — только видео + закадровый голос
3. Загрузить видео или сгенерировать через встроенный AI (промт из сценария)
4. Вставить текст диктора (будет звучать как закадровый голос)

**Для сцены типа СЛАЙД:**
1. Добавить новую сцену
2. Загрузить слайд, без аватара
3. Вставить текст диктора как закадровый голос

### Шаг 16. Переходы и финальная настройка

- Добавить плавные переходы между сценами (Magic Match или Dissolve)
- Проверить, что все тексты вставлены корректно
- Включить автоматические субтитры (Captions) — русский язык
- Установить качество: **1080p** (или 4K для премиум)

### Шаг 17. Рендер

- Нажать **Submit / Generate**
- Время рендера: ~5–15 минут для 10-минутного видео
- Дождаться уведомления

## ЭТАП 6: КОНТРОЛЬ КАЧЕСТВА ВИДЕО (QA)

### Шаг 18. Чек-лист просмотра

Просмотреть видео целиком. Проверить по пунктам:

| # | Критерий | ✅/❌ |
|---|----------|------|
| 1 | Аватар двигает губами синхронно с речью | |
| 2 | Нет искажений лица аватара (глитчи, артефакты) | |
| 3 | Текст диктора соответствует утверждённому сценарию | |
| 4 | Слайды читаемы, текст не перекрыт аватаром | |
| 5 | B-roll вставки релевантны тому, что говорит диктор | |
| 6 | Переходы между сценами плавные, нет обрывов звука | |
| 7 | Субтитры корректны (нет ошибок в транскрипции) | |
| 8 | Общий хронометраж соответствует плану (±20%) | |
| 9 | Единый стиль: один аватар, один голос, один шаблон слайдов | |
| 10 | Нет технических артефактов (чёрные кадры, зависания) | |

### Шаг 19. Работа над ошибками

| Проблема | Причина | Решение |
|----------|---------|---------|
| Аватар плохо синхронизирует губы | Длинные предложения, сложные слова | Разбить предложения, упростить термины |
| Аватар "замирает" или глитчит | Слишком длинная сцена (>90 сек) | Разбить сцену на две |
| Голос звучит роботизированно | Неудачный выбор TTS голоса | Сменить голос или использовать voice cloning |
| B-roll не соответствует теме | Плохой промт для генерации | Переписать промт, быть конкретнее |
| Слайд нечитаем за аватаром | Аватар перекрывает контент | Увеличить свободную зону на слайде, уменьшить аватар |
| Субтитры с ошибками | Автотранскрипция | Отредактировать вручную в редакторе HeyGen |

### Шаг 20. Финальный экспорт

- Скачать .mp4 в максимальном качестве
- Сохранить в папку проекта: `Курсы/[Название курса]/Лекция N — [Тема].mp4`
- При необходимости — мелкие правки в CapCut/Premiere (обрезка, вставка заставки)

### Точка согласования №4: ПРИЁМКА ВИДЕО

## ТИПИЧНЫЕ ОШИБКИ (чего НЕ делать)

| Ошибка | Почему плохо | Что делать вместо |
|--------|-------------|-------------------|
| Начать работу без брифа | Клиент скажет "я имел в виду другое" | Всегда начинать с брифа |
| Не согласовать текст с клиентом | Термины будут неправильные, факты неточные | Отправить на проверку эксперту клиента |
| Сразу рендерить без согласования сценария | Перерасход credits на переделки | Показать сценарий + тестовый ролик 15 сек |
| Один аватар на все курсы разных клиентов | Клиенты увидят одинаковые лекции | Менять аватар/лук под каждый курс |
| Игнорировать B-roll, делать только "говорящую голову" | Скучно, внимание падает после 2 минут | Чередовать типы сцен каждые 30–90 сек |
| Длинные сцены (>90 сек одного типа) | Аватар глитчит, зритель засыпает | Разбивать, чередовать |
| Не проверить липсинк на русском | Русский — не приоритет HeyGen, бывают артефакты | Тестировать голос на сложных словах перед полным рендером |
