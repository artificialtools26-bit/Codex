# ПАЙПЛАЙН ГЕНЕРАЦИИ ЛЕКЦИЙ — М-Платформа
## Версия 1.0

---

## ОБЩАЯ СХЕМА

```
Глава учебника (.docx)
       ↓
[ЭТАП 1] Подготовка текста — Claude (словарь + предобработка)
       ↓
[ЭТАП 2] Генерация презентации — NotebookLM + 1-2 итерации правок в нём же
       ↓
[ЭТАП 3] Верификация — Claude (текст) + человек (картинки) + ручная доделка текста
       ↓
[ЭТАП 4] Озвучка — ElevenLabs (голос) или HeyGen (AI-аватар)
       ↓
[ЭТАП 5] Финальная сборка — CapCut / FFmpeg
       ↓
Готовая видеолекция (.mp4)
```

## ЭТАП 1: ПОДГОТОВКА ТЕКСТА (до NotebookLM)

**Цель:** подготовить материал так, чтобы NotebookLM выдал минимум ошибок.

### 1.1 Предобработка источника

Загрузить главу учебника в Claude и дать промт:

```
Ты — редактор образовательного контента. Подготовь текст главы
для передачи в AI-генератор презентаций.

Задачи:
1. Убери сложные таблицы — перепиши их как списки или короткие абзацы
2. Раскрой все аббревиатуры при первом упоминании: РД (ручная дуговая),
   МП (механизированная плавящимся электродом) и т.д.
3. Замени сложные составные термины на простые формулировки где возможно,
   но НЕ МЕНЯЙ стандартизованные термины из ГОСТов
4. Разбей текст на логические блоки по 1-2 абзаца — каждый блок = 1 слайд
5. В конце каждого блока добавь строку: [ВИЗУАЛ: описание того,
   что должно быть на слайде — схема/таблица/фото чего именно]

Источник:
<текст главы>
```

### 1.2 Создание словаря терминов

Из подготовленного текста извлечь все технические термины и создать файл:

```
Извлеки все технические термины, аббревиатуры и сложные слова
из текста. Для каждого укажи:
- Термин как пишется
- Как правильно произносится (по слогам с ударением)
- Расшифровка (если аббревиатура)
- Английский эквивалент (если есть)

Формат:
| Термин | Произношение | Расшифровка | EN |
```

**Пример результата:**

| Термин | Произношение | Расшифровка | EN |
|--------|-------------|-------------|-----|
| РД | эр-дэ | ручная дуговая | MMA |
| МП | эм-пэ | механизированная плавящимся | MIG |
| МПГ | эм-пэ-гэ | механизированная порошковой | MAG |
| РАД | эр-а-дэ | ручная аргонодуговая | TIG |
| аргонодуговая | ар-го-но-ду-ГО-вая | — | — |
| кристаллизация | крис-тал-ли-ЗА-ция | — | crystallization |

→ Сохранить как `словарь_{тема}.md`
→ Этот же словарь пойдёт в ElevenLabs (этап 4)

### 1.3 Создание инструкции для NotebookLM

Загрузить в NotebookLM как источник вместе с главой:

```
ИНСТРУКЦИЯ ДЛЯ ГЕНЕРАЦИИ ПРЕЗЕНТАЦИИ

Тема: [название]
Аудитория: [кто будет смотреть]

Требования к слайдам:
- Используй ТОЛЬКО информацию из приложенной главы учебника
- Не выдумывай примеры, кейсы, компании
- Все аббревиатуры писать в формате: РД / MMA (с расшифровкой)
- Таблицы и сравнения приветствуются
- Для иллюстраций: используй СХЕМЫ и ДИАГРАММЫ,
  НЕ пытайся рисовать реалистичные изображения людей или оборудования
- Стиль: технический/инженерный (чертёжная сетка, чистый дизайн)

Запрещено:
- Изображения людей (сварщиков, рабочих)
- Реалистичные изображения оборудования
- Декоративные иконки без смысла
- Метафоры и аналогии, которых нет в тексте
```

## ЭТАП 2: ГЕНЕРАЦИЯ В NOTEBOOKLM

### 2.1 Загрузка материалов
1. Загрузить подготовленный текст (из этапа 1.1)
2. Загрузить инструкцию (из этапа 1.3)
3. Загрузить системные правила (файл `Правила генерации NOTEBOOK LM.docx`)
4. **Если в оригинале есть картинки/чертежи** — загрузить также оригинальный .docx файл главы. NotebookLM получит и структурированный текст из п.1, и оригинальные схемы/чертежи из исходника. Это важно для технических тем, где чертежи несут смысловую нагрузку (поперечные сечения швов, размеры кромок и т.д.)

### 2.2 Генерация
- Запросить создание презентации
- Скачать результат (.pptx)

### 2.3 Итерации правок в NotebookLM (1-2 раунда)

NotebookLM может сам подправить результат — стоит использовать это до ручной работы.

**Раунд 1 — быстрый просмотр:**
- Пролистать слайды, отметить явные проблемы
- Попросить NotebookLM переделать конкретные слайды:
  «Слайд 3 — убери картинку, сделай вместо неё таблицу сравнения»
  «Слайд 5 — текст про РАД неточный, перефразируй по источнику»

**Раунд 2 (если нужен):**
- Если после первого раунда осталось 2+ проблемы — ещё одна итерация
- Если всё ещё плохо → переходить к ручным правкам на этапе 3

**Правило:** больше 2 итераций в NotebookLM не делать — дальше быстрее руками

## ЭТАП 3: ВЕРИФИКАЦИЯ И ПРАВКИ

**Цель:** найти и исправить ошибки ДО озвучки.

### 3.1 Автоматическая проверка (Claude)

Извлечь текст из сгенерированной презентации и прогнать через Claude:

```
Ты — технический верификатор образовательного контента.

У тебя есть:
1. Оригинальный текст учебника (источник истины)
2. Текст из сгенерированной презентации

Проверь презентацию по критериям:

КРИТИЧНОСТЬ 2 (блокер — нельзя выпускать):
- Фактические ошибки (термины, цифры, классификации)
- Неправильные аббревиатуры
- Информация, которой НЕТ в источнике (галлюцинации)
- Орфографические ошибки в терминах

КРИТИЧНОСТЬ 1 (нужно исправить):
- Неточные формулировки, искажающие смысл
- Пропущенные важные моменты из источника

КРИТИЧНОСТЬ 0 (желательно исправить):
- Стилистические замечания
- Избыточные упрощения

Для каждой ошибки укажи:
- Слайд №
- Что написано (ошибка)
- Как должно быть (по источнику)
- Критичность (0/1/2)
```

### 3.2 Проверка визуала и текста (человек)

**ВАЖНО:** NotebookLM выдаёт слайды как PNG-картинки, а не как редактируемые элементы.
Это значит: нельзя просто взять и заменить одну иконку на слайде.

**Стратегии исправления слайдов-картинок:**

**Способ 1 — Перегенерация в NotebookLM (лучший для плохих картинок)**
Если на слайде дичь (сварка без перчаток, нереальная горелка) →
вернуться в NotebookLM и попросить переделать конкретный слайд:
«Слайд 5 — не рисуй людей и оборудование, сделай только текстовую схему»
Это самый чистый способ — получаешь новый PNG в том же стиле.

**Способ 2 — Наложение в Canva / Figma (быстрый фикс)**
1. Импортировать слайд-PNG как фон в Canva
2. Закрыть плохую картинку белым прямоугольником
3. Поставить поверх нормальное фото из банка
4. Экспортировать обратно как PNG
→ Быстро, но может немного отличаться по стилю

**Способ 3 — Замена целого слайда (для критичных случаев)**
Если слайд совсем плохой — сделать его с нуля в Canva/Figma
в похожем стиле (чертёжная сетка, тот же шрифт, те же цвета).
Один-два таких слайда на лекцию — допустимо.

**Текстовые правки на PNG-слайдах:**
- Мелкие опечатки → Способ 2 (закрыть + написать правильно поверх в Canva)
- Серьёзные текстовые ошибки → Способ 1 (перегенерировать слайд)

**Правило:** если на лекцию из 7-10 слайдов нужно переделать больше 3 →
проще перегенерировать всю презентацию с уточнённым промтом

### 3.3 Порядок правок
1. Сначала попробовать исправить через NotebookLM (способ 1)
2. Что не исправилось — закрыть/наложить в Canva (способ 2)
3. Если совсем не спасти — пересобрать слайд (способ 3)
4. Если ошибок критичности 2 больше двух → вернуться к этапу 2 полностью

## ЭТАП 4: ОЗВУЧКА (ElevenLabs или HeyGen)

### 4.1 Подготовка текста для озвучки

Из проверенной презентации создать скрипт озвучки через Claude:

```
Создай текст для озвучки по каждому слайду презентации.

Правила:
1. Один слайд = один аудиосегмент (30-90 секунд)
2. Говори от лица лектора, обращайся к аудитории на «вы»
3. Не читай слайд дословно — РАСКРЫВАЙ тезисы
4. Все аббревиатуры произноси по словарю:
   РД = "эр-дэ", MMA = "эм-эм-эй", не "эм-эм-ай"
5. Ударения в сложных словах расставь через знак `:
   аргонодуго́вая, механизи́рованная, кристаллиза́ция
6. Тон: профессиональный лектор, понятно и без воды
7. Не добавляй информацию, которой нет в источнике

Формат вывода:
---СЛАЙД 1---
[текст для озвучки]
---СЛАЙД 2---
[текст для озвучки]
...
```

### 4.2 Настройка ElevenLabs

**Выбор голоса:**
- Язык: русский
- Тип: мужской или женский профессиональный голос
- Рекомендуемые модели: `eleven_multilingual_v2` (лучшее качество для русского)

**Загрузка словаря произношений:**
В ElevenLabs → Projects → Pronunciation Dictionary → загрузить словарь из этапа 1.2

Формат словаря для ElevenLabs (файл .pls или через интерфейс):
```
РД → эр дэ
МП → эм пэ
МПГ → эм пэ гэ
РАД → эр а дэ
MMA → эм эм эй
MIG → миг
MAG → маг
TIG → тиг
```

### 4.3 Генерация аудио

**Вариант А — через интерфейс:**
1. Зайти в ElevenLabs → Speech Synthesis
2. Выбрать голос и модель
3. Вставить текст каждого слайда отдельно
4. Скачать .mp3 файлы: `slide_01.mp3`, `slide_02.mp3`, ...

**Вариант Б — через API (для автоматизации):**
```bash
npm install elevenlabs
```

```javascript
const { ElevenLabsClient } = require("elevenlabs");
const fs = require("fs");

const client = new ElevenLabsClient({ apiKey: "YOUR_API_KEY" });

async function generateAudio(text, slideNum) {
  const audio = await client.textToSpeech.convert("VOICE_ID", {
    text: text,
    model_id: "eleven_multilingual_v2",
    voice_settings: {
      stability: 0.6,        // чуть ниже = живее
      similarity_boost: 0.8,  // высокое = ближе к оригиналу голоса
      style: 0.3,             // лёгкая экспрессия
    },
  });

  const fileName = `slide_${String(slideNum).padStart(2, "0")}.mp3`;
  const chunks = [];
  for await (const chunk of audio) {
    chunks.push(chunk);
  }
  fs.writeFileSync(fileName, Buffer.concat(chunks));
  console.log(`Готово: ${fileName}`);
}
```

### 4.4 Альтернатива: HeyGen (AI-аватар)

Если клиент хочет «говорящую голову» — используем HeyGen:

1. Загрузить тот же текст озвучки (из 4.1)
2. Выбрать аватар (или создать кастомный)
3. Загрузить словарь произношений
4. Генерировать видео по сегментам

**Когда ElevenLabs, когда HeyGen:**

| Критерий | ElevenLabs (голос) | HeyGen (аватар) |
|----------|-------------------|-----------------|
| Качество речи | Отличное | Хорошее |
| Визуал | Нет (только голос) | Аватар на экране |
| Стоимость | Дешевле | Дороже |
| Скорость | Быстрее | Медленнее |
| Когда использовать | Преза + закадровый голос | Клиент явно просит аватар |

**Рекомендация:** по умолчанию ElevenLabs + слайды. HeyGen — как опция за доплату.

### 4.5 Борьба с произношением в HeyGen (русский язык)

> **Проблема:** HeyGen плохо работает с русскими ударениями и техническими терминами. Это нормально — русский не приоритетный язык движка. Ниже — хаки от простого к сложному.

**Хак 1 — Фонетическая подмена (первое, что пробуем)**

Переписать слово так, как оно должно звучать. HeyGen читает побуквенно — помоги ему:

| Проблема | Что пишем в скрипте |
|----------|-------------------|
| «бУквенно» (ставит ударение не туда) | `БУквенно` или `БУ-квен-но` |
| «сварнЫх» (ставит ударение не туда) | `свар-НЫХ` |
| «нахлёсточные» (читает без ё) | `нахлЁсточные` |
| ГОСТ 5264-80 (читает как число) | `гост пятьдесят два шестьдесят четыре тире восемьдесят` |
| α, β (пропускает или коверкает) | `альфа`, `бета` |

**Хак 2 — Разбивка на слоги дефисами**

Если заглавные буквы не помогают — разбей слово:
```
проплавление → про-плав-ЛЕ-ние
притупление → при-туп-ЛЕ-ние
криволинейный → кри-во-ли-НЕЙ-ный
электрододержатель → элек-тро-до-дер-ЖА-тель
```

**Хак 3 — Замена на синоним или перефразирование**

Если слово совсем не поддаётся — переписать фразу:

| Не получается | Замена |
|---------------|--------|
| «буквенно-цифровой код» | «код из букв и цифр» |
| «нахлёсточное соединение» | «соединение внахлёст» |
| «криволинейный скос» | «скос по дуге» или «изогнутый скос» |

**Хак 4 — Озвучка в ElevenLabs + загрузка аудио в HeyGen (крайний случай)**

Если после хаков 1–3 слово всё равно звучит плохо:
1. Озвучить весь текст (или проблемный сегмент) в **ElevenLabs** — там словарь произношений работает корректно
2. Скачать .mp3
3. В HeyGen: создать сцену → вместо текста загрузить аудиофайл → аватар синхронизирует губы по аудио

Это самый надёжный способ, но самый долгий. Использовать только когда хаки 1–3 не помогли.

**Порядок действий при проблемном слове:**
```
Шаг 1: Попробовать ЗАГЛАВНЫЕ буквы на ударном слоге
Шаг 2: Не помогло → разбить на слоги дефисами
Шаг 3: Не помогло → заменить на синоним/перефразировать
Шаг 4: Не помогло → озвучить сегмент в ElevenLabs, загрузить аудио в HeyGen
```

### 4.6 Проверка аудио / видео
- [ ] Прослушать каждый сегмент
- [ ] Проверить произношение терминов
- [ ] Если ошибка → поправить текст → перегенерировать только этот сегмент

## ЭТАП 5: ФИНАЛЬНАЯ СБОРКА

### Вариант А — CapCut (простой, бесплатный)

1. Импортировать слайды как изображения (экспортировать из .pptx → PNG)
2. Расположить на таймлайне
3. Наложить аудиодорожки на каждый слайд
4. Синхронизировать длительность слайда = длительность аудио
5. Добавить переходы (fade, 0.3 сек)
6. Включить автосубтитры (CapCut делает бесплатно)
7. Экспорт → MP4, 1080p

### Вариант Б — FFmpeg (автоматизация для 200 лекций)

```bash
# 1. Экспорт слайдов из .pptx в PNG
# (через LibreOffice или python-pptx)
libreoffice --headless --convert-to pdf presentation.pptx
pdftoppm -png -r 300 presentation.pdf slide

# 2. Создать видео из слайдов + аудио
# Для каждого слайда:
ffmpeg -loop 1 -i slide_01.png -i slide_01.mp3 \
  -c:v libx264 -tune stillimage -c:a aac \
  -shortest -pix_fmt yuv420p segment_01.mp4

# 3. Склеить все сегменты
# Создать файл segments.txt:
# file 'segment_01.mp4'
# file 'segment_02.mp4'
# ...
ffmpeg -f concat -safe 0 -i segments.txt \
  -c copy lecture_final.mp4
```

### Вариант В — Python-скрипт (полная автоматизация)

Скрипт, который берёт папку со слайдами (PNG) + папку с аудио (MP3)
и выдаёт готовое видео. Можем написать когда будете готовы.
